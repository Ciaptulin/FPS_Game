# Git 提交与更新·极简笔记

---

### 1. 核心概念图（先记这三个区）
理解这个，命令就不会乱：
> **工作区**（你改代码的地方） ➡️ `git add` ➡️ **暂存区**（临时存放） ➡️ `git commit` ➡️ **本地仓库**（历史记录） ➡️ `git push` ➡️ **远程仓库（GitHub）**

---

### 2. 日常标准提交流程（三步走）
这是最常用的命令，每次更新代码必敲：

```bash
# 1. 查看当前修改状态（红色=未添加，绿色=已暂存）
git status

# 2. 添加所有改动到暂存区（常用 . 或 -A）
git add .        # 添加当前目录所有变动（含新增、修改、删除）

# 3. 生成一个版本记录（务必写清楚改了什么）
git commit -m "feat: 添加了用户登录功能"

# 4. 推送到 GitHub 远程仓库
git push origin main   # 注意：main 也可能是 master，看你默认分支
```

---

### 3. `git add` 的三种区别（别再乱用了）
| 命令             | 作用范围                               | 适用场景                             |
| :--------------- | :------------------------------------- | :----------------------------------- |
| `git add .`      | **当前目录**及子目录下的所有变动       | 最常用，只要你在项目根目录执行       |
| `git add -A`     | **整个仓库**所有目录的变动（包含上级） | 想在根目录外执行时用，保险起见用这个 |
| `git add 文件名` | 仅添加指定单个文件                     | 只想提交某一个文件时用               |

---

### 4. 提交信息规范（推荐格式）
好的 `commit` 信息便于回溯，推荐使用 **类型(范围): 描述** 格式：

- `feat: 新增XXX接口` （新功能）
- `fix: 修复登录态过期bug` （修复问题）
- `docs: 更新README文档` （仅文档修改）
- `style: 调整按钮间距` （格式/样式，不涉及逻辑）
- `refactor: 重构工具类方法` （代码重构）

---

### 5. 远程同步与冲突解决（避坑必看）
如果 `git push` 时报错（被拒绝），说明远程有别人先提交了：

```bash
# 方案一（推荐）：拉取远程代码并变基，保持提交记录线性整洁
git pull origin main --rebase

# 方案二（传统）：拉取并自动合并
git pull origin main

# 解决完冲突后（若有），继续推送
git push origin main
```

> **冲突解决小贴士**：若 `--rebase` 后出现冲突，手动修改文件，然后执行：
> `git add .` ➡️ `git rebase --continue` ➡️ 最后 `git push`

---

### 6. 常用查看与撤销命令（救命指令）
| 需求                                 | 命令                             |
| :----------------------------------- | :------------------------------- |
| 查看简洁提交历史（看哈希值）         | `git log --oneline`              |
| 查看某文件改了什么                   | `git diff 文件名`                |
| **撤销**暂存区（add多了想撤回）      | `git reset HEAD 文件名`          |
| **修改**最近一次提交的信息           | `git commit --amend -m "新信息"` |
| **回退**到上一个版本（保留代码修改） | `git reset --soft HEAD~1`        |
| 放弃所有未提交的本地修改（危险！）   | `git checkout .`                 |

---

### 7. 新电脑/新项目首次连接 GitHub
如果是第一次提交，需要先关联远程地址：

```bash
# 1. 初始化本地仓库
git init

# 2. 关联远程仓库（URL换成你自己的）
git remote add origin https://github.com/Ciaptulin/FPS_Game.git

# 3. 第一次推送（-u 会建立默认关联，下次直接 git push 即可）
git push -u origin main
```
